# 完成前验证

**这是强制步骤，所有 agent 完成任务前必须执行此验证。**

## 验证优先级

```
1. plan 是否实现    ← 最重要！计划里写的都做到了吗？
2. README 是否正确  ← 重要！文档与代码一致吗？
3. 代码能否跑通     ← 基本！能运行吗？
4. 语法错误         ← 最基本！有 bug 吗？
```

---

## 第一步：检查 plan 是否实现

### 1. 找到 plan 文件

检查以下位置：
- `plan.md`
- `PLAN.md`
- `docs/plan.md`
- `plans/*.md`
- `.opencode/plans/*.md`

### 2. 提取计划要求

从 plan 文件中提取：
- 功能列表
- 技术方案
- 验收标准
- TODO 列表

### 3. 逐一验证实现

对于每个计划项：
- 阅读相关代码确认实现
- 运行相关功能确认工作
- 记录未实现或部分实现的项目

### 4. 输出验证结果

```
### Plan 验证结果

| 计划项 | 状态 | 验证 |
|--------|------|------|
| 用户登录 | ✅ 已实现 | auth/login.ts |
| 数据导出 | ❌ 未实现 | 未找到代码 |
| API 限流 | ⚠️ 部分实现 | 只有 IP 限流 |

未实现: N 项
部分实现: N 项
完全实现: N 项
```

**如果有未实现项，不能声称任务完成！**

---

## 第二步：检查 README 是否正确

### 1. 检查 README.md 存在

### 2. 验证文档内容

- 项目描述是否准确
- 安装步骤能否执行
- 使用示例能否运行
- API 文档是否完整

### 3. 运行文档中的命令

```bash
# 安装
bun install

# 运行
bun run dev

# 示例
bun run example
```

### 4. 对比文档与代码

- 文档说的 API 是否存在
- 参数是否正确
- 返回值是否正确

```
### README 验证结果

| 检查项 | 状态 |
|--------|------|
| 安装步骤 | ✅ |
| 运行命令 | ❌ 报错 |
| API 文档 | ⚠️ 缺少 2 个 |

问题: ...
```

**如果 README 与代码不一致，必须修正！**

---

## 第三步：验证代码能否跑通

### 1. 安装依赖

```bash
bun install
```

### 2. 运行测试

```bash
bun test
```

### 3. 构建项目

```bash
bun run build
```

### 4. 启动服务（如适用）

```bash
bun run dev
bun run start
```

```
### 运行验证结果

| 检查项 | 状态 | 结果 |
|--------|------|------|
| 依赖安装 | ✅ | 148 packages |
| 测试 | ❌ | 3/10 失败 |
| 构建 | ✅ | dist/ 生成 |

失败原因: ...
```

**如果代码跑不通，不能声称任务完成！**

---

## 第四步：检查语法错误

### 1. 类型检查

```bash
bun run typecheck
```

### 2. Lint 检查

```bash
bun run lint
```

### 3. 代码审查

检查：
- 空指针风险
- 边界条件
- 异常处理
- 资源泄漏

```
### 语法验证结果

| 检查项 | 状态 | 数量 |
|--------|------|------|
| 类型 | ❌ | 5 错误 |
| Lint | ✅ | 0 警告 |

错误详情: ...
```

**如果有语法错误，必须修复！**

---

## 最终判定

```
## 验证总结

### 状态: ✅ 通过 / ❌ 未通过

### Plan 实现: N/M 项完成
### README 正确: ✅/❌
### 代码跑通: ✅/❌
### 语法正确: ✅/❌

### 问题列表
1. ...
2. ...

### 可以声称完成吗？
✅ 可以 - 所有验证通过
❌ 不可以 - 存在未解决的问题
```

---

## 强制规则

1. **必须执行全部四步验证**
2. **任何一项不通过都不能声称完成**
3. **必须如实报告所有问题**
4. **未实现的 plan 项必须列出**
5. **README 不一致必须修正或报告**
