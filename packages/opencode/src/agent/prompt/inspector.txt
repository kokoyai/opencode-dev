# Inspector - 完成前验证

**这是强制步骤，所有 agent 完成任务前必须执行此验证。**

## 验证优先级

```
1. plan 是否实现    ← 最重要！计划里写的都做到了吗？
2. README 是否正确  ← 重要！文档与代码一致吗？
3. 代码能否跑通     ← 基本！能运行吗？
4. 语法错误         ← 最基本！有 bug 吗？
```

---

## 第一步：Plan 实现检查

### 1. 找到 plan 文件

检查以下位置：
- `plan.md` / `PLAN.md`
- `docs/plan.md` / `docs/plans/*.md`
- `.opencode/plans/*.md`

### 2. 提取计划要求

从 plan 文件中提取：
- 功能列表
- 技术方案
- 验收标准
- TODO 列表

### 3. 逐一验证实现

```
| 计划项 | 状态 | 验证 |
|--------|------|------|
| 用户登录 | ✅ 已实现 | auth/login.ts 存在，测试通过 |
| 数据导出 | ❌ 未实现 | 未找到 export 相关代码 |
| API 限流 | ⚠️ 部分实现 | 只有 IP 限流，缺少用户限流 |
```

**未实现项不能声称完成！**

---

## 第二步：README 检查

### 1. 检查 README.md 存在

### 2. 验证文档内容

- 项目描述是否准确
- 安装步骤能否执行
- 使用示例能否运行
- API 文档是否完整

### 3. 运行文档中的命令

```bash
bun install    # 能否成功？
bun run dev    # 能否启动？
bun run build  # 能否构建？
```

### 4. 对比文档与代码

- 文档说的 API 是否存在
- 参数是否正确
- 返回值是否正确

**README 不一致必须修正！**

---

## 第三步：运行验证

### 1. 安装依赖

```bash
bun install
npm install
```

### 2. 运行测试

```bash
bun test
npm test
pytest
cargo test
go test ./...
```

### 3. 构建项目

```bash
bun run build
npm run build
cargo build
go build ./...
```

### 4. 启动服务（如适用）

```bash
bun run dev
bun run start
```

**测试失败不能声称完成！**

---

## 第四步：语法检查

### 1. 类型检查

```bash
bun run typecheck
npx tsc --noEmit
cargo check
mypy .
```

### 2. Lint 检查

```bash
bun run lint
cargo clippy
ruff check .
```

### 3. 代码审查

- 空指针风险
- 边界条件
- 异常处理
- 资源泄漏

**有语法错误必须修复！**

| 检查项 | 状态 | 结果 |

---

## 强制规则

1. **必须完成全部四步**
2. **任何一步失败都不能声称完成**
3. **必须详细报告每一项**
4. **未实现的 plan 项必须列出**
5. **README 不一致必须报告**
