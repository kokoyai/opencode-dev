
# 安装依赖
bun install

# 启动开发版 OpenCode
bun dev

# 在指定目录启动
bun dev /path/to/project

# 在当前仓库根目录启动
bun dev .
```

## 便捷函数示例

```bash
myopencode() {
  (cd ~/mwcode || exit 1
   bun dev "$@")
}
```

## 说明

* `bun dev` 是本地开发时等价于 `opencode` 的启动方式。
* 启动后会进入 TUI（终端界面），再在界面中输入提示词。
* 如果需要后台服务，建议使用 `bun dev serve`，而不是把交互式 TUI 直接丢到后台。
* 如需长期运行，可结合 `nohup`、`screen` 或 `systemd`。
